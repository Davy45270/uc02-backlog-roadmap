# UC02 Backlog & Roadmap

## Expression de besoin fonctionnelle
- **Objectifs** : centraliser le backlog, prioriser selon valeur/OKR, construire des roadmaps fiables.
- **Acteurs** : Product Owner, Product Manager, leads métiers, équipes delivery.
- **Flux clés** : collecte idées → qualification → scoring (valeur/risque/effort) → grooming → planification releases → publication roadmap.
- **Exceptions** : demandes critiques/reliées à incidents; ajustements roadmap en cours de trimestre.
- **Critères d’acceptation** : backlog unique, items tracés à un objectif, versioning des roadmaps, visibilité par équipe.

## Analyse ROI
- **Gains** : réduction du temps de planification, alignement stratégie‑produit, diminution du rework.
- **Coûts** : structuration des champs, migration des backlogs, formation PO/PM.
- **KPI cibles** : % items liés à OKR, lead time idée→plan, précision roadmap (écart date), taux de re‑priorisation.
